## 1. 背景与问题

### 1.1 背景

近五年全球轨道发射进入「双强驱动」时代：

- **SpaceX（美国商业航天代表）** 以高频发射、一级回收、星链（Starlink）大规模组网重塑了发射经济学。
- **中国** 以国家队 + 商业航天双轨并进，长征系列、新一代运载火箭与低轨星座（如国网 / 千帆等）持续扩容。

公开报道与统计口径大致显示（数据会漂移，产品内需可更新）：

| 维度（约 2025 年量级） | 美国 / SpaceX | 中国 |
|------------------------|--------------|------|
| 年度轨道发射次数 | 美约 180–193 次；SpaceX Falcon 约 165 次 | 约 92–93 次 |
| 全球占比 | 美中合计约占全球 ~88% | 同上 |
| 可回收一级 | Falcon 9 常态化着陆与复用（单箭 30+ 次飞行量级） | 可回收验证与部分商业尝试（尚未形成 SpaceX 级常态复用） |
| 巨型星座 | Starlink 已部署数千–上万量级 | 国网 / 千帆等规划与在轨建设中 |

用户关心的是：**马斯克/SpaceX 在天上「发了多少、回收了多少、还要发多少」**，以及 **中国发射与星座规模如何对照**——而不是读枯燥 PDF 或 Wiki 表格。

### 1.2 要解决的问题

1. **信息碎片化**：发射、回收、星座、规划分散在 Wiki、SpaceNews、官方通告等多源。
2. **时间维度缺失**：用户需要「过去 → 现在 → 未来规划」一条时间轴叙事。
3. **对比维度不全**：只比「次数」不够，还需回收、复用、在轨卫星、成功率等。
4. **观感平淡**：航天题材适合科幻级视觉与动效，当前多数图表站不够沉浸。

### 1.3 产品定位一句话

> 一个**科幻美学的中美航天成绩对照台**：用沉浸式前端展示 SpaceX（及美国）与中国在**火箭发射、回收复用、卫星星座**上的**历史、现状与未来规划**。

## 2. 目标与非目标

### 2.1 目标（Goals）

| # | 目标 | 成功标准 |
|---|------|----------|
| G1 | 一眼看懂中美/SpaceX 核心差距与各自优势 | 首屏 5 秒内完成「谁在发射、谁在回收、谁的星在天上」认知 |
| G2 | 覆盖过去 / 现在 / 未来三条时间线 | 至少支持年粒度历史 + 当年进度 + 规划目标三态 |
| G3 | 科幻感强、可分享 | 深色星空 UI、动效、可截图/可分享链接；移动端可用 |
| G4 | 数据可维护 | 数据与 UI 解耦；JSON/YAML 可改，无需改代码即可刷新数字 |
| G5 | 中文优先、专业可信 | 中文界面；关键指标附数据来源与「截至日期」 |

### 2.2 非目标（Non-Goals · v1 不做）

- 不做实时发射直播 / 发射倒计时（可列为 v2）
- 不做完整任务级数据库（单次发射详情可外链，不自建百科）
- 不做账号体系、评论、UGC
- 不做政治立场宣传；产品定位为**公开数据可视化 + 科普**
- 不接入付费 API 作为 v1 强依赖（可后续接入）
- 不覆盖俄罗斯、欧空局等第三方完整对照（可预留「全球」入口）

## 3. 用户与场景

### 3.1 目标用户

| 画像 | 诉求 |
|------|------|
| 航天/科技爱好者 | 好看、好懂、可分享的对照图 |
| 内容创作者 | 截图做图、引用数字、讲故事 |
| 产品/投资观察者 | 快速了解发射频次、复用、星座规模量级 |
| 学生/科普 | 时间线 + 术语解释 |

### 3.2 核心使用场景

1. **打开即对比**：用户进入页面，默认看到中美双栏对照仪表盘。
2. **时间穿梭**：拖动时间轴，看 2015→2025 发射曲线如何拉开。
3. **深入某一维**：点「回收」看着陆次数、复用次数、单箭飞行纪录；点「星座」看 Starlink vs 中国星座。
4. **看未来**：切换到「规划」面板，看 SpaceX 年度目标、Starship 路线、中国星座规划数量级。
5. **移动端转发**：微信/朋友圈可读的竖屏摘要卡。

## 6. 数据需求

### 6.1 设计原则

1. **口径优先于精度幻觉**：标明「轨道发射尝试 / 成功」「国家口径 vs 公司口径」。
2. **SpaceX ≠ 美国全部**：总览需同时呈现「美国合计」与「SpaceX 单独」两层（可切换）。
3. **未来数据标注为规划**：规划目标 ≠ 承诺，UI 上用虚线/「规划」徽章。
4. **中国数据多源**：国家航天局、航天科技/科工新闻、SpaceNews 等交叉核对。
5. **可版本化**：每次数据文件带 `schemaVersion` + `updatedAt` + `sources[]`。

### 6.2 核心实体

```
CountrySnapshot
  country: "US" | "CN"
  year: number
  orbitalAttempts: number
  orbitalSuccesses: number
  notes?: string

OperatorSnapshot          # e.g. SpaceX, CASC, commercial CN
  operatorId: string
  year: number
  launches: number
  successRate?: number

ReuseMetrics              # primarily SpaceX
  operatorId: "spacex"
  asOf: date
  boosterLandings: number
  consecutiveLandings?: number
  maxFlightsPerBooster: number
  reuseRate?: number      # fraction of flights with flown boosters

Constellation
  id: "starlink" | "guowang" | "qianfan" | ...
  country: "US" | "CN"
  deployedLifetime: number      # cumulative launched or active — specify field
  activeInOrbit?: number
  plannedFull?: number
  asOf: date

Milestone
  date: string
  side: "US" | "CN" | "BOTH" | "GLOBAL"
  title: string
  description: string
  category: "launch" | "reuse" | "constellation" | "policy"

FuturePlan
  side: "US" | "CN" | "SPACEX"
  horizon: string           # "2026" | "2030" | ...
  metric: string
  value: number | string
  source: string
  confidence: "official" | "reported" | "estimate"
```

### 6.3 v1 建议指标清单（页面必显）

**发射**

- 历年轨道发射次数：美国 / 中国 / SpaceX
- 累计轨道发射（可选起始年：2006 SpaceX 首飞 / 1970 中国首星）
- 当年 YTD 与目标（若有）

**回收**

- SpaceX 一级着陆成功次数（累计）
- 单枚助推器最高飞行次数
- 中国：首次海上回收验证、可重复使用火箭关键试验节点（叙事为主）

**卫星**

- Starlink：累计发射 / 在轨活跃（注明估算）
- 中国：年度入轨卫星数量级 + 主要星座规划规模
- （可选）全球在轨活跃卫星中美占比示意

**未来**

- SpaceX：当年发射目标区间、Starship 运营化节点（公开表述）
- 中国：星座规划星数、可回收火箭部署节奏（公开规划）

### 6.4 数据种子（PRD 参考量级 · 实现时以 JSON 为准并标注来源）

> 以下为 2026-07 前后公开报道的**量级参考**，非最终入库数据。

| 指标 | 参考量级 | 备注 |
|------|----------|------|
| 2025 SpaceX Falcon 轨道发射 | ~165 | 多家媒体一致报道量级 |
| 2025 中国轨道发射 | ~92–93 | 不同来源 70/92/93 需统一口径 |
| 2025 美国轨道发射 | ~181–193 | 含美国公司境外发射等口径差异 |
| Falcon 9 家族累计发射 | ~670+（2026 中） | 持续增长 |
| Falcon 单箭最高飞行次数 | 30+ 量级 | 纪录不断刷新 |
| Starlink | 数千–上万在轨量级 | 需区分 launched vs active |
| 中国低轨星座规划 | 数千–上万星量级（分星座） | 分「国网 / 千帆」等条目 |

**v1 数据策略**：人工整理 `data/seed.json`，页脚写「数据截至 YYYY-MM-DD，仅供科普展示」。

### 6.5 推荐公开数据源

- SpaceNews / NASASpaceflight / Spaceflight Now
- Wikipedia: Falcon 9 launches、2025/2026 in spaceflight
- Jonathan McDowell / planet4589 统计
- 中国航天科技集团 / 国家航天局新闻稿
- UCS Satellite Database（在轨卫星，更新频率较低）
- SpaceX 官网 / 官方 X 账号声明（规划类）

## 7. 交互与视觉（科幻前端）

### 7.1 视觉关键词

- **Deep Space Navy / Void Black** 背景
- **Neon Cyan（SpaceX/美）** vs **Crimson Gold（中国）** 双色体系
- 细线网格、HUD 边框、扫描线、微粒星尘
- 字体：标题用几何无科技感无衬线；数字用等宽/表格数字
- 动效：缓入、视差星空、KPI 跳动、图表描边生长；**避免眩晕与过重粒子**

```
└─────────────────────────────────────────────────────────────┘
```

### 7.3 交互原则

1. **先结论后细节**：KPI → 图表 → 下钻。
2. **对称对比**：同一指标左右同时出现，避免一边有一边无（无数据则显示「暂无/验证中」）。
3. **规划态视觉降权**：未来数据用虚线、半透明、「规划」标签。
4. **可访问性**：对比度达标；动效尊重 `prefers-reduced-motion`。
5. **性能**：首屏 LCP 目标 < 2.5s（本地静态资源）；图表懒加载。

### 7.4 文案语气

- 中性、克制、科普向；避免「碾压」「完胜」等情绪化标题。
- 可用科幻修辞包装 UI（「轨道观测站」「遥测面板」），但数字区保持冷静。

## 9. 内容与合规

1. **商标与肖像**：可用公开事实描述 SpaceX / 马斯克相关成就；避免未授权商标 logo 作商业暗示；马斯克肖像非必须，优先火箭/轨道意象。
2. **免责声明**：「本站为科普可视化，数据来自公开报道，可能存在口径差异与延迟，不构成投资或政策建议。」
3. **政治中立**：对比维度限定航天工程指标，不做意识形态结论。
4. **引用**：关键数字旁附 source id，Sources 页列出链接。

## 11. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| 数据口径打架（70 vs 93 等） | 可信度下降 | 统一采用「轨道发射尝试」并标注来源；冲突处写 note |
| 在轨卫星数难以精确 | 误导 | 显示区间或「约」；区分 launched / active |
| 3D 过重拖垮移动端 | 体验差 | 默认 2D/CSS；3D 仅桌面或开关 |
| 规划目标频繁变化 | 过时 | `updatedAt` + 季度人工复核清单 |
| 敏感解读 | 传播风险 | 中性文案 + 免责声明 |
